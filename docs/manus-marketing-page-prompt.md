# Manus AI Prompt: Concentrade Marketing Page Rebuild

## Project Context

You are rebuilding the marketing/landing page for **Concentrade**, a professional trading journal SaaS application. The app helps traders log, analyze, and improve their trading through AI-powered behavioral insights, performance analytics, and emotional intelligence tracking. The primary competitor we are positioning against is **Tradezella**.

The app is built with **Next.js (App Router)**, **Tailwind CSS v4**, **TypeScript**, **Framer Motion**, and **shadcn/ui** components. The page lives at `/app/page.tsx` and uses a shared `<MarketingNav />` component and `<ConcentradeLogo />` component (these already exist and should be reused).

---

## Objective

Build a **highly compelling, conversion-optimized marketing page** that persuades traders — particularly those currently using Tradezella or spreadsheets — to switch to Concentrade. The page must feel like it was designed by a seasoned growth-focused design team, not generated by AI.

## Design Constraints (CRITICAL)

### Color Palette — Maximum 4-5 Colors
1. **Primary Background**: `slate-950` (#020617) — ultra-deep navy-black. NOT pure black.
2. **Primary Brand/Accent**: `indigo-500` / `indigo-600` (#6366f1 / #4f46e5) — used for CTAs, highlights, and interactive elements.
3. **Success/Positive**: `emerald-400` / `emerald-500` (#34d399 / #10b981) — used for win rates, profit indicators, positive data.
4. **Danger/Negative**: `rose-500` or warm coral (#f43f5e) — used for loss indicators, pain points, and "chaos" messaging.
5. **Neutrals**: `slate-300` through `slate-800` for text hierarchy and card backgrounds.

### What to AVOID
- **NO purple gradients** as primary design elements. The infamous `from-purple-600 to-indigo-600` gradient screams "AI-generated SaaS page." If purple appears, it should only be a very subtle accent, never a gradient hero or CTA.
- **NO gradient text** on body copy or sub-headlines. Gradient text is acceptable ONLY on the hero headline keywords ("Chaos" and "Profits").
- **NO floating abstract blobs**, decorative gradient circles, or blurry orb backgrounds. These are AI design cliches.
- **NO generic stock photos** of people pointing at screens or shaking hands.
- **NO overuse of glow effects**. A single subtle `shadow-indigo-500/20` on a card is fine. Stacking multiple glows looks artificial.
- **NO "glassmorphism everywhere"** approach. Use `backdrop-blur` sparingly — only on the nav and maybe one featured card.
- **NO emoji as icons**. Use Lucide React icons exclusively.
- **NO more than 2 font families**. Use the system font stack or a single Google Font (Inter, Plus Jakarta Sans, or similar professional sans-serif).

### What TO DO
- Use **solid, flat card backgrounds** with subtle borders (`border-slate-800`) and slight background tints (`bg-slate-900/50`).
- Use **generous whitespace**. Sections should breathe. Minimum `py-24` between major sections.
- Use **real data mockups** (win rates, P&L numbers, trade counts) inside feature preview cards to look authentic.
- Use **micro-interactions** with Framer Motion: subtle fade-in-up on scroll, gentle hover lifts on cards (`whileHover={{ y: -4 }}`), and staggered animations on lists.
- Typography hierarchy: hero headline 5xl-7xl bold, section headlines 4xl-5xl bold, body text xl with `leading-relaxed`, labels/captions xs-sm with `tracking-widest uppercase`.

## Technical Requirements

### Framework & Dependencies
- Next.js 15 App Router
- TypeScript
- Tailwind CSS v4 (uses `@theme inline` in globals.css, no tailwind.config.js)
- Framer Motion for animations
- Lucide React for icons
- shadcn/ui Button component

### Component Architecture
- The page should be a **client component** (`"use client"`) since it uses Framer Motion
- Import and reuse `<MarketingNav />` from `@/components/marketing/marketing-nav`
- Import and reuse `<ConcentradeLogo />` from `@/components/concentrade-logo`
- Import `Button` from `@/components/ui/button`
- Import `Link` from `next/link`
- Import `motion` from `framer-motion`
- Import `cn` from `@/lib/utils`

### Animation Guidelines
- Use `motion.div` with `initial={{ opacity: 0, y: 16 }}` and `animate={{ opacity: 1, y: 0 }}` for entrance animations
- Stagger delays by 0.1s increments for sequential elements
- Use `whileHover={{ y: -4 }}` for card hover effects
- Keep all transition durations between 0.3s-0.6s
- Use `whileInView` with `viewport={{ once: true }}` for scroll-triggered animations so they only fire once

### Responsive Design
- Mobile-first approach
- Hero: single column on mobile, full-width on desktop
- Pain points: stack vertically on mobile, 2-col grid on `md:`
- Features: stack vertically on mobile, 2-col alternating on `lg:`
- Testimonials: single column on mobile, 3-col on `md:`
- Pricing: single column on mobile, 3-col on `md:`

### Routes
- `/signup` — registration page
- `/login` — login page
- `/demo` — interactive demo
- `#features` — scroll to features section
- `#pricing` — scroll to pricing section
- `#testimonials` — scroll to testimonials section
- `/privacy` — privacy policy
- `/terms` — terms of service

## Emotional Messaging Framework

The page follows this psychological arc:

1. **Hook** (Hero) — "Your trading is chaos. We make it profitable." Immediate identification with the pain.
2. **Agitate** (Pain Points) — "You're not failing because of strategy — you're failing because of data." Deepen the problem.
3. **Solution** (Features) — "Here's exactly how we fix it." Show concrete, data-rich feature previews.
4. **Proof** (Testimonials) — "Don't take our word for it." Real stories from real traders.
5. **Compare** (Table) — "See how we stack up." Rational comparison that confirms the emotional decision.
6. **Convert** (Pricing + Final CTA) — "Start free, scale when ready." Remove friction.

Every section should feel like it was written by someone who trades, not someone who builds SaaS landing pages. Use trading terminology naturally: "expectancy," "edge," "setups," "prop firm," "P&L," "revenge trading," "tilt."
